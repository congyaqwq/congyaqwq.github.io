MVC和MVVM的区别

MVVM:M指后端数据传递，V指所看到的页面，VM是核心，M转为V实现方式为数据绑定；V转为M，实现方式为DOM事件监听
MVC:通过C的来承上启下
区别:MVVM只是弱化了C的概念，VM作用是解析数据,VM抽离C中展示的业务逻辑，而不是代替，视图操作等业务应该还是放在C中实现

meta标签三要素

content

必选，为了定义http-equiv或name属性相关的元信息。便于SEO查找信息与分类
前提name=”viewport”
width:可视区域宽度，值可以为数字或者关键字device-width
height:与width相同
initial-scale:页面首次显示是可视取悦的缩放级别
maximum-scale,minimum-scale:可视区域缩放级别
user-scalable:是否可以对也面进行缩放，no禁止

name

可选，用于描述网页，可选值: author,description,keyword,generator

http-equiv

相当于http的头文件，向浏览器返回有用信息，以帮助正确和精确显示内容。值可以有content-type,expires,refresh

懒加载和预加载

懒加载：

img元素，如果没有src属性，浏览器就不会发出请求去下载图片，使用js设置了图片路径，浏览器才会发送请求去下载图片，要用的时候再设置。

实现原理：

纯粹的延迟加载：使用setTimeOut和setInterval进行加载延迟
首先，不要放图片地址到src属性中；
页面加载完成后，根据scrollTop判断图片是否在用户视野中，如果在，则将属性放到src中
在滚动事件中重复判断图片是否进入视野，如果进入，将属性放到src中

优点

页面加载速度快，减轻服务器压力，节约流量，用户体验好

意义

目的主要是作为服务器前端的优化，减少请求数或延迟请求数

预加载

提前加载图片，当用户需要查看时可直接从本地缓存中读取

意义

牺牲服务器性能，换取更好的用户体验，使用户操作时得到最好的体验

实现方式：

实现预载的方法非常多，比如：用CSS和JavaScript实现预加载；仅使用JavaScript实现预加载；使用Ajax实现预加载。
常用的是new Image();设置其src来实现预载，再使用onload方法回调预载完成事件。只要浏览器把图片下载到本地，同样的src就会使用缓存，这是最基本也是最实用的预载方法。当Image下载完图片头后，会得到宽和高，因此可以在预载前得到图片的大小(方法是用记时器轮循宽高变化)。

[].constructor === Array

3.使用Object.prototype.toString.call(arr)

function isArray(o){
return Object.prototype.toString.call(o);
}

4.ES5方法

Array.isArray(arr)

css中的子元素中的百分比（%）到底是谁的百分比？

子元素height和width百分比

相对于子元素的直接父元素，width相对父元素width，height相对于父元素height

top,bottom,left,right

如果设置top，bottom为百分比，则相对于非static定位的父元素的高度
如果设置left和right为百分比，则相对于直接非static定位的父元素的宽度

padding，margin

无论水平或垂直，都相对于直接父元素的width，与height无关

js假性值

1.false (布尔型)
2.null (用于定义空的或者不存在的引用)
3.undefined (未定义值)
4.0 (数值型)
5."'' (空字符串) (字符型)
6.NaN
